# Contribuir a VectorLab

Gracias por tu interés en contribuir. VectorLab es un proyecto de código abierto y toda ayuda es bienvenida.

## Cómo contribuir

1. **Reportar bugs** — Abre un [issue](https://github.com/Jeremias0618/VectorLab/issues) describiendo el problema y pasos para reproducirlo.
2. **Sugerir mejoras** — Ideas nuevas, mejoras de UX o de exportación (PNG, SVG, WebP) también pueden ir como issues.
3. **Enviar código** — Fork del repo, crea una rama, haz tus cambios y abre un Pull Request.

## Flujo de trabajo

1. Haz fork del repositorio.
2. Crea una rama desde `main`: `git checkout -b feature/mi-mejora` o `fix/descripcion-del-bug`.
3. Realiza tus cambios y asegúrate de que `npm run lint` pase.
4. Haz commit con mensajes claros (ej: "Añadir exportación a WebP").
5. Abre un Pull Request describiendo qué cambias y por qué.

## Estándares de código

- Usamos **TypeScript** y **React** con las reglas de ESLint del proyecto.
- Ejecuta `npm run lint` antes de enviar tu PR.
- Mantén los componentes y funciones razonablemente pequeños y legibles.

## Áreas donde puedes ayudar

- Editor de iconos SVG
- Editor de logos
- Exportación a PNG, SVG y WebP (tamaños y calidad)
- Plantillas de iconos y logos
- Mejoras de accesibilidad y diseño responsive
- Documentación y traducciones

Si tienes dudas, abre un issue y lo comentamos. ¡Gracias!
